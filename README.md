# Beurreland.cc

Welcome to Beurreland, a sharp-edged retro website dedicated to the sacred religion of butter, questionable lore, and internet nonsense done with love.

This project is for random internet visitors and anyone who still believes websites should have personality.

## What Is This?

Beurreland is a vanilla PHP website with:

- A public guestbook (because shouting into the void is fun)
- A contact form that sends HTML emails to the Butter Gods
- A small account/API key panel
- Public API endpoints for guestbook data
- Retro visual style, marquee energy, and zero modern corporate vibes

Main feature?

- Yes.

## Stack

- PHP (vanilla)
- MySQL
- JavaScript + CSS (retro-first)
- PHPMailer
- HTMLPurifier
- vlucas/phpdotenv

## Quick Start ("do your best" edition)

### 1. Install dependencies

```bash
composer install
```

### 2. Create the database and import schema

```bash
mysql -u YOUR_DB_USER -p YOUR_DB_NAME < database.sql
```

### 3. Create `.env` at project root

Use these variables:

```env
DATA_HOST=127.0.0.1
DATA_NAME=beurreland
DATA_USER=your_db_user
DATA_PASS=your_db_password

MAIL_HOST=smtp.example.com
MAIL_USER=noreply@example.com
MAIL_PASS=your_mail_password
MAIL_PORT=587
MAIL_TEST_RECIPIENT=you@example.com

# Optional / legacy note:
# AUTH_TOKEN is referenced in comments in the guestbook API,
# but current auth uses API keys stored in DB tables.
# AUTH_TOKEN=optional
```

### 4. Run locally

```bash
php -S 127.0.0.1:8000 -t public
```

Then open:

`http://127.0.0.1:8000`

## Main Routes

- `/` home page + lore
- `/guestbook.php` guestbook UI
- `/contact.php` contact form
- `/panel.php` account and API key management
- `/api/v1/guestbook.php` guestbook API (GET/POST/DELETE)
- `/api/v1/tor.php` returns tor hostname value from server file

## Project Structure

```text
public/           Web root (pages, API, assets)
src/php/          Shared backend logic (DB, mail, bbcode)
inc/              Shared layout pieces (sidebar/footer)
database.sql      SQL schema
```

## Security Notes

- Uses CSRF tokens on forms
- Uses honeypot fields and basic anti-spam timing checks
- Sanitizes HTML content with HTMLPurifier
- Uses DB-backed API keys for protected guestbook API actions
